Gulp & Grunt

…or how I gulped and grunted my way through this presentation.

What the heck are Gulp & Grunt?!

Essentially, they are javascript task runners for streamlining development and optimizing site performance, among (a ton of) other things.

With either tool, almost any repetitive task can be automated:

  • code minification/beautification
  • code quality analysis
  • image optimization/compression
  • css vendor prefixing
  • unit testing
  • deployment
  • much, much more

The tools are quite similar in some respects:

  • both use node.js
  • rely on npm (node package manager)
  • use javascript to write tasks
  • …and that's it

They are actually quite different under the hood.

Under the Hood

Grunt

Task Configuration

  • Requires extensive configuration of gruntfile.js
  • structured around built-in tasks
    • concat, init, lint, min, qunit, server, test, watch
  • typically requires defining I/O for each task
  • tasks can be extended by plugins
  • plugins can be very complex, and may handle many tasks

Under the Hood

Gulp

Task Configuration

  • emphasis on code
  • no built in tasks
  • uses node's streams
    • Streams are a method of combining myriad tiny transform operations into a 'pipeline'
  • tasks are created by piping in processes (think chaining jQuery methods)

Under the Hood

Grunt

Task Execution

  • each task is executed sequentially:
    • open file
    • execute the current process
    • save changes
    • close the current processed file for the next process to access it
  • use of intermediary temp files/disk writes

Under the Hood

Gulp

Task Execution

  • again, node streams
  • tasks are executed with maximum concurrency using Orchestrator
  • executes in memory
  • no temp files
  • uses a virtual file format called Vinyl
    • a metadata object format with 2 main attributes (path and contents)
    • examples: local file system, files on S3, FTP, Dropbox, Box, node streams

ECOSYSTEM

Grunt

Plugins & Community

  • since late 2011
  • 5189 plugins
  • has a larger user base and more mature community
  • Github metrics
    • watchers: 597, stars: 10,016, forks: 1,204

ECOSYSTEM

Gulp

Plugins & Community

  • since mid 2013
  • 1881 plugins
  • has a (seemingly) smaller user base
  • based on Github metrics, is more popular than Grunt
    • watchers: 803, stars: 16,615, forks: 1,770
  • community-built tasks work well with each other
  • plugin philosophy: “do one thing, and do it well.”

Error Handling

Grunt

Grunt already has robust error handling


Gulp

Gulp's team doesn't deny that built-in error management "sucks"


Errors are either a result of failed tasks or failed pipeline portions

Examples

Sample gruntfile.js

A basic CSS minifier, grunt flavored

module.exports = function(grunt) {
  // Display task-execution time in the CLI
  require('time-grunt')(grunt);
  // Configuration
  grunt.initConfig({
    pkg: grunt.file.readJSON('package.json'),
    // Combine CSS files in src folder
    concat_css: {
      dist: {
        src: ['src/*.css'],
        dest: 'css/app.css'
      }
    },
    // Minify css
    cssmin: {
      target: {
        // Write the minified file in the css directory
        files: [{
          'css/app.min.css': ['css/app.css']
        }]
      }
    }
  });
  // Load plugins
  grunt.loadNpmTasks('grunt-concat-css');
  grunt.loadNpmTasks('grunt-contrib-cssmin');
  // Execute with `grunt` command
  grunt.registerTask('default', ['concat_css', 'cssmin']);
}

Examples

Sample gulpfile.js

A basic CSS minifier, gulp flavored

// Load plugins
var gulp = require('gulp');
var concatenate = require('gulp-concat-css');
var minify = require('gulp-cssmin');
var duration = require('gulp-duration');
// Configuration
gulp.task('default', function() {
  gulp.src('src/*.css')
  // Combine css files in src folder
  .pipe(concatenate('app.min.css'))
  // Minify CSS
  .pipe(minify())
  // Display task-execution time in CLI
  .pipe(duration('Execution Time: '))
  // Save minified file in the css folder
  .pipe(gulp.dest('css/'));
});
// Execute with `gulp` command

Down the road

Grunt

grunt-next (v1 alpha) uses Orchestrator.

  • No temp files
  • concurrent task execution
  • increased speed

Gulp

  • better error handling
  • see many improvements

Which is better?

To grunt or to gulp?

It depends.


Chances are, the project will determine which you use.


Given time, you will likely use both.
